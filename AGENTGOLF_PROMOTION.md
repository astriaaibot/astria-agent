## Frames Strategy (Cost-Effective)

### Frame 1: Registration

**What it does:**
- Asks for name, email, agent idea
- Stores in Supabase
- Sends confirmation

**Flow:**
```
Clawdbot post: "Register for AgentGolf â†’ [button]"
User clicks "Register"
Frame shows: "What's your name?" (text input)
Next: "What's your email?" (text input)
Next: "What's your agent idea?" (text area)
Submit â†’ "You're registered! âœ…"
```

**Backend (cost-effective):**
```python
@app.route("/frame/register", methods=["GET"])
def register_frame_initial():
    """Initial registration frame"""
    return {
        "version": "vNext",
        "image": "https://yourserver.com/register-1.png",
        "input": {"text": "Your name..."},
        "buttons": [{"label": "Next", "action": "post"}]
    }

@app.route("/frame/register", methods=["POST"])
def register_frame_submit():
    """Handle submission"""
    data = request.json
    name = data["untrustedData"]["inputText"]
    fid = data["untrustedData"]["interactor"]["fid"]
    username = data["untrustedData"]["interactor"]["username"]
    
    # Store in Supabase
    supabase.table("agentgolf_registrations").insert({
        "fid": fid,
        "username": username,
        "name": name,
        "timestamp": datetime.now(),
    }).execute()
    
    return {
        "version": "vNext",
        "image": "https://yourserver.com/registered.png",
        "buttons": [{"label": "View Entries", "action": "link", "target": "https://miniapp.example.com"}]
    }
```
